## Rate Limit問題

Nix Flakesは依存関係の解決のためにGitHub APIを叩くので、何度も試行錯誤していると、rate limitに引っかかる場合がある。特にinputsに入れてる依存の数が増えるほど、GitHubのAPIの叩く数が増えるので引っかかりやすくなる。引っかかるとこんな感じのメッセージがでる。

```sh
error: unable to download 'https://api.github.com/repos/minad/corfu/commits/HEAD': HTTP error 403

       response body:

       {
         "message":"API rate limit exceeded for 133.200.204.224. (But here's the good news: Authenticated requests get a higher rate limit. Check out the documentation for more details.)",
         "documentation_url":"https://docs.github.com/rest/overview/resources-in-the-rest-api#rate-limiting"
       }
```

素直に https://docs.github.com/rest/overview/resources-in-the-rest-api#rate-limiting をみればよい。そこからlinkされているhttps://docs.github.com/en/rest/using-the-rest-api/rate-limits-for-the-rest-api?apiVersion=2026-03-10 に以下のように書いてある。

>Primary rate limit for unauthenticated users
>You can make unauthenticated requests if you are only fetching public data. Unauthenticated requests are associated with the originating IP address, not with the user or application that made the request.
>
>The primary rate limit for unauthenticated requests is 60 requests per hour.
>
>Primary rate limit for authenticated users
>You can use a personal access token to make API requests. Additionally, you can authorize a GitHub App or OAuth app, which can then make API requests on your behalf.
>
>All of these requests count towards your personal rate limit of 5,000 requests per hour. Requests made on your behalf by a GitHub App that is owned by a GitHub Enterprise Cloud organization have a higher rate limit of 15,000 requests per hour.

要するに、GitHubのAPIレートリミットは未認証60回/時間だが、認証を通せば最低でも5000回/時間に増える。エラーメッセージに"But here's the good news: Authenticated requests get a higher rate limit."と書いてあるとおりである。5000回/時間なら大抵の場合は十分だろう。

個人用アクセストークン（Personal access token, PAT）の発行は今のところ以下の手順で可能:

1. GitHubの Settings > Developer settings > Personal access tokens > Tokens (classic)を選択。
2. "Generate new token (classic)" から発行。

PATの権限は選択可能だが。公開リポジトリの情報を取得するだけであれば、追加権限は不要だった。

これをnixから使う方法はいくつかある。

## GitHub CLIを使う方法

これは[uma-chan](https://zenn.dev/i9wa4)さんから聞いた方法である。

[GitHub CLI](https://docs.github.com/ja/github-cli/github-cli/about-github-cli)をつかっている場合は、`gh auth token`でPATが取得できる。したがって、`nix flake update`するときにこのPATを与えれば認証された状態でアクセスできる。

```sh
access_token=$(gh auth token)
nix flake update --access-tokens "github.com=$access_token"
```

この方法はファイルにPATを直に書かなくて良いという理由で優れている。

## nix.confに設定する方法

以下で紹介されていた方法である。

https://zenn.dev/kawarimidoll/articles/0a4ec8bab8a8ba

https://discourse.nixos.org/t/flakes-provide-github-api-token-for-rate-limiting/18609/9


nixはデフォルトで`~/.config/nix/nix.conf`と`/etc/nix/nix.conf`にある`access-tokens`を参照してくれる。認証タイプの指定も必要で次のように書けば良い。

```nix
access-tokens = github.com=ghp_abcd1234
```

環境変数`NIX_USER_CONF_FILES`を使えば、複数の設定ファイルは指定できるので、ウェブ公開しているなどの理由で`nix.conf`にを書けない場合も、他のファイルを指定できる。

```sh
export NIX_USER_CONF_FILES="${XDG_CONFIG_HOME}/nix/nix.conf:${XDG_CONFIG_HOME}/nix/local.conf"
```

こうすれば`${XDG_CONFIG_HOME}/nix/local.conf`に書いた内容もnixが見てくれる。

home-managerを使っている場合は`nix.extraOptions`を使えば`/etc/nix/nix.conf`に追記できる。agenixによる暗号化と組み合わせた方法が以下のサイトで紹介されていたのでリンクしておく。

https://apribase.net/2025/02/20/agenix/

## 環境変数で指定する方法

環境変数でも設定できる。

```sh
export GITHUB_TOKEN=abcd1234
```

nixはこの変数を読んでくれるようだ。

## for GitLab

GitLabでも同様の問題があり、同様の緩和策が取れるようだ。自分の手元ではまだ検証していない。
